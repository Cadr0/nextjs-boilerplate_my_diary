# Diary AI - Handoff Context

Этот файл нужен как короткий, но полный контекст для нового чата с ИИ.
Его можно отправлять целиком, чтобы следующий ИИ быстро понял проект, текущее состояние и направление работы.

## 1. Что это за продукт

`Diary AI` - это личная система наблюдения за собой.
Пользователь не просто пишет заметки, а постепенно собирает:

- запись дня
- личные метрики
- задачи рядом с днём
- историю дней
- аналитику
- AI-анализ по одному дню или периоду

Главная идея продукта:

- интерфейс должен быть спокойным, чистым и понятным
- продукт должен помогать человеку замечать динамику и улучшать себя
- нельзя превращать его в перегруженный dashboard

Ключевой UX-смысл:

- сначала простой и красивый вход
- потом личное рабочее пространство
- потом постепенное наращивание функций

Проект строится поэтапно.
Сейчас нельзя расползаться в сложные разделы раньше времени.

Правильный порядок:

## 5. Текущее направление по дизайну

Пользователь очень чувствителен к визуальному качеству.
Нельзя делать "типичный AI-лендинг", перегруженные блоки и случайные декоративные элементы.

Нужный визуальный характер:

- calm tech
- premium minimal
- mobile-first
- крупная, выразительная типографика
- мягкие поверхности
- аккуратные тени
- светлая спокойная палитра
- понятная визуальная иерархия

Чего пользователь не хочет:

- перегруженные hero-блоки
- лишние плашки и пояснения
- сайдбары в auth
- перегруженный mockup, который спорит с формой
- дизайн ради дизайна без пользы

Формула, которую нужно держать в голове:

- минимум отвлекающих элементов
- максимум ясности
- продукт для людей, которые хотят меняться, отслеживать показатели и улучшать их

## 12. Важные продуктовые принципы

### 1. Mobile-first

Продукт должен в первую очередь хорошо ощущаться на телефоне.

### 2. Один экран - одна главная мысль

Если экран можно упростить, его нужно упростить.

### 3. История должна быть исторически корректной

Если метрики меняются со временем, старые записи не должны ломаться.
Для этого дальше потребуется snapshot-подход.

### 4. AI - не декорация

AI должен давать полезный результат по дню или периоду, а не быть случайной кнопкой.

## 13. Что делать дальше

Если auth-входной поток станет визуально стабильным, следующий правильный шаг:

1. довести главную `/` до финального первого впечатления
2. затем перейти к основному экрану `/diary`
3. построить сильный рабочий интерфейс дневника и метрик
4. только потом расширять историю, аналитику и AI

## 14. Что нужно от следующего ИИ

Следующий ИИ должен:

- сначала изучить текущие файлы входного потока
- не ломать рабочую auth-логику
- улучшать интерфейс очень осторожно
- мыслить как senior UI/UX designer и product engineer
- держать фокус на простоте, порядке и mobile-first
- не добавлять визуальный шум
