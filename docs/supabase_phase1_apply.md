# Supabase Phase 1 - Как снять блокер `user_id + RLS`

Сейчас сайт уже работает с auth, но дневник ожидает защищенную схему БД.

Текущая ошибка:

- `добавьте user_id в daily_entries и включите RLS`

## Что запускать в Supabase SQL Editor

### Шаг 1. Базовая схема и политики

Откройте SQL Editor и целиком выполните файл:

- `supabase/sql/2026-03-19_phase1_private_cabinet.sql`

Этот SQL:

- создает `profiles`
- добавляет `user_id` в `daily_entries`
- включает RLS для `profiles` и `daily_entries`
- создает базовые политики на свои данные
- создает trigger на автоматическое создание профиля при новом signup

### Шаг 2. Backfill старых записей

Если в `daily_entries` уже есть старые записи, выполните файл:

- `supabase/sql/2026-03-19_phase1_backfill_existing_entries.sql`

Порядок:

1. найти свой `auth.users.id`
2. подставить его вместо `<YOUR_USER_UUID>`
3. привязать старые строки к пользователю
4. проверить дубли по `entry_date`
5. только после этого включать уникальность по (`user_id`, `entry_date`)

## Что должно получиться после применения

- вход через Google работает
- `/diary` открывает только личные записи
- новые записи сохраняются с `user_id`
- RLS не дает читать и редактировать чужие строки

## Что делать сразу после SQL

1. открыть сайт
2. снова зайти в `/diary`
3. создать новую запись дня
4. проверить, что запись сохраняется без ошибки
