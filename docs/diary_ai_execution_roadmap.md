# Diary AI - Короткий roadmap на ближайшие шаги

## Текущий фокус

Сейчас закрываем первый блок:

- стартовая страница
- вход через Google OAuth
- callback и SSR-сессия
- приватный маршрут `/diary`
- базовый визуальный фундамент под дальнейшие разделы

## Порядок работ

### 1. Public + Auth Foundation

- публичный лендинг
- отдельная страница входа
- Supabase SSR auth
- proxy и callback

Результат:

- пользователь понимает, что это личный продукт, а не тестовая форма

### 2. Private Data Foundation

- `profiles`
- `user_id` во всех личных сущностях
- строгие RLS-политики
- защита API-роутов

Результат:

- у каждого пользователя появляется действительно приватный кабинет

### 3. Diary Constructor

- выбор пресета при первом входе
- гибкие метрики
- порядок метрик
- улучшенный daily flow

Результат:

- дневник становится личным рабочим инструментом, а не просто формой

### 4. History + Analytics

- история дней
- snapshot-подход для старых записей
- первые графики и фильтры

Результат:

- у пользователя появляется видимая динамика и доверие к данным

### 5. AI Workspace

- AI по одному дню
- AI по периоду
- сохраненные результаты анализа
- шаблоны промтов

Результат:

- AI становится полезной функцией, а не декоративным дополнением

## Правило на ближайшие итерации

Двигаемся только блоками:

1. закрываем auth и приватность
2. потом закрываем личную модель данных
3. потом расширяем сам дневник
4. только после этого открываем историю, аналитику и AI глубже
